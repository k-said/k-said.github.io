Buffer Zone Tool

What is this tool?

The Buffer Zone Tool computes pesticide buffer zones around agricultural fields. Buffer zones are setback distances that must be maintained between a treated field and nearby vulnerable areas (waterbodies, forests, residential areas, etc.) based on product-specific regulations.

The tool combines three key inputs:

  • Field boundary — the polygon of the field to be treated
  • Vulnerable areas — nearby sensitive features (auto-detected from OpenStreetMap by default, or uploaded / drawn manually)
  • Slope data — terrain slope around the field (auto-derived from Copernicus DEM 25 m by default, or uploaded as GeoJSON)

These inputs are combined with product-specific rules (drift reduction requirements, buffer distances) to produce the final buffer zone geometries and a summary of applicable constraints.

Step-by-Step Guide

① Select a Country

Choose your region (Germany, US, or Brazil) at the top. This loads country-specific products, rules, and vulnerable area types.

② Define Field Boundary

Your field is shown on the map by default. To change it:

  • Upload a .geojson file using the file picker
  • Draw a polygon or rectangle directly on the map using the toolbar (top-left)
Tip: You can edit or delete the field polygon after drawing it.

③ Vulnerable Areas

Choose how vulnerable areas near the field are determined:

  • Auto-detect (OSM) — The API queries OpenStreetMap for waterbodies, forests, etc. No action needed.
  • Upload GeoJSON — In the OSM panel, use the "Upload Vulnerable Areas GeoJSON" file picker to supply your own vulnerable areas. This overrides the auto-detection from OSM.
  • Custom (Draw/Upload) — Switch to the Custom tab to draw polygons on the map or upload a GeoJSON. Select the type for each area (Aquatic, Terrestrial, etc.).

To draw a custom area: select the type from the dropdown, click Draw Polygon, then draw on the map. You can add multiple areas of different types.

Tip: You can also add new custom vulnerable area types using the "+ Add" controls.

④ Slope Data (optional)

By default the API derives slope from the Copernicus DEM (25 m) around the field. If you have your own slope data:

  • Open the "Slope Data (optional)" section in the sidebar
  • Upload a .geojson file. This will be sent to the API instead of the auto-derived slope.
  • Use the Clear button to revert to the default DEM-based slope.

⑤ Select Products

Products are listed for your selected country. Move products from Available to Selected using the arrow buttons. Only rules for selected products will be used in the calculation.

Tip: Select multiple products by holding Ctrl (or Cmd on Mac) while clicking.

⑥ Advanced Settings (optional)

Click "Show / Hide Advanced Settings" to edit:

  • Rules table — CSV data mapping products to buffer distances and drift reductions. Upload a custom CSV or edit directly.
  • Parameters — JSON controlling OSM tag mappings and column names. Upload or edit.

☞ Calculate & Results

Click Calculate Buffer Zones. The API will return:

  • Buffer zones — coloured polygons on the map (green = low drift, red = high drift reduction)
  • Vulnerable areas — the detected or supplied areas shown by type
  • Slope data — if slope-based rules exist

Use the layer control (top-right of map) to toggle layers. Download results as GeoJSON using the buttons below the results summary.

Tip: Click "Edit Vuln. Areas & Re-submit" to refine the detected areas and recalculate.

Data Defaults

Vulnerable Areas OpenStreetMap (auto-detected) Override via upload
