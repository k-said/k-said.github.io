🌾 Yield Map Prediction (Prototype)

Contact: Said Kharbouche | For more info: Confluence Documentation

📖 Help & User Guide

1. Quick Start

  1. Draw or upload a field boundary on the map (use the polygon tool on the left of the map, or upload a GeoJSON file).
  2. Set parameters in the sidebar: crop, country, planting date, inference date (or days-after-planting), and the prediction target.
  3. Click ▶️ Start Polling to submit the request and wait for results.
  4. The yield map will appear on the map once processing completes.

2. Configuration Fields

Field Description
Field-Id / Field Name A name or CWO field ID. When no boundary is drawn, the system uses this ID + the CWO Token to fetch the field boundary from the Cropwise Operations API.
CWO Token Cropwise Operations API token. Only required if you rely on field-ID lookup instead of drawing/uploading a boundary.
Country Country where the field is located (US or AR). Auto-detected if a boundary is provided.
Crop Crop type: Corn or Soy.
Prediction Target Which yield variable to predict. See the unit info box for details on each target.
Planting Date Date the crop was planted.
Date Mode Inference Date: pick a calendar date; Days After Planting: specify how many days after planting to run inference.
Map Background Switch between Satellite, Map (OpenStreetMap), or Hybrid views.
Map Mode Relative: colours cells as a percentage of the top-N percentile value; Absolute: colours by raw yield value. Adjust the top percentile with the number input.

3. Boundary Tools

  • Draw: Use the polygon tool on the map (top-left toolbar) to draw a field boundary.
  • Upload: Click 📁 Upload Boundary to load a GeoJSON or JSON file containing a FeatureCollection.
  • Clear: Remove the current boundary.
  • Reset: Reload the built-in default boundary.
  • Download: Export the current boundary as a GeoJSON file.

4. How Polling Works

After clicking ▶️ Start Polling, the first request is sent to the API. Because the Lambda function may take longer than the API Gateway timeout (29 s), the first request may return a timeout — this is expected. The system then polls every 30 seconds (first wait is 60 s) for up to 5 minutes until a result is available.

  • Running means the server is still computing.
  • Success means the yield map is ready and will be displayed automatically.
  • Failed means an error occurred — check the Message Log for details.

5. Understanding the Results

  • Yield Map: Each coloured cell (10 m × 10 m) shows the predicted yield value.
  • Legend: Appears in the bottom-right corner. In Relative mode it shows percentages; in Absolute mode it shows raw yield values.
  • Stats: The sidebar shows summary statistics (count, mean, std, min, max).
  • NDVI Chart: Displays the mean NDVI time series (with ±1 std shading) used during inference.
  • Info Panel: Shows metadata returned by the model (adjusted dates, area, etc.).
  • Unit Toggle: Switch between SI/Metric and US units (bu/ac). Conversion is applied to the displayed values and legend.

6. Downloads

  • ⬇️ Download Boundary: Exports the field boundary as .geojson.
  • ⬇️ Download Yield Map: Exports the full prediction GeoJSON (enabled after a successful prediction).

7. Tips & Troubleshooting

  • If you see 502 / 504 errors in the log, the server is likely still processing — just wait for the next poll.
  • Maximum field size is 400 ha (configured on the server).
  • The prediction cache is active for a limited time. To force a fresh run on the same field, wait for the cache to expire or change the field ID.
  • Toggle Show Yield Map and Show Boundary checkboxes to show/hide layers.
